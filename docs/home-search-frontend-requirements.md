# 商城首页与搜索结果页前端开发要求（PC 优先）

## 1. 文档目标

本文档用于指导商城首页与搜索结果页的前端页面开发，页面整体风格参考京东 PC 商城的布局与视觉节奏，但不要求像素级复刻。

本次页面开发统一复用现有搜索接口。首页通过空搜索参数获取推荐商品，搜索结果页基于关键词与筛选条件检索商品，并且必须支持基于上一次搜索结果返回的相关品牌、相关属性、相关分类继续缩小检索范围。

## 2. 接口与数据约定

### 2.1 搜索接口

- 请求方式：`POST`
- 公开接口：`/public/product/search`
- 返回结构：`R<ProductSearchResult>`

前端只调用公开接口，不调用内部接口 `/product/search`。

### 2.2 通用响应结构

接口返回统一包装对象，前端需要先判断请求是否成功，再使用 `data`。

```json
{
  "code": "SUCCESS",
  "msg": "success",
  "data": {
    "products": [],
    "total": 0,
    "pages": 0,
    "pageNum": 0,
    "brands": [],
    "attrs": [],
    "catalogs": []
  }
}
```

前端处理规则：

- 仅在接口成功时渲染最新结果。
- 接口失败时展示错误提示，并保留上一次成功结果。
- 前端无需传递 `userId`，该信息由后端自动绑定。

### 2.3 请求参数定义

搜索参数来自 `ProductSearchParam`，前端必须按以下规则组装：

| 字段 | 类型 | 说明 | 前端要求 |
| --- | --- | --- | --- |
| `keyword` | `String` | 搜索关键字 | 首页搜索框、搜索结果页顶部搜索框使用 |
| `catalogId` | `Long` | 商品分类 ID | 单选分类筛选 |
| `sort` | `List<String>` | 排序规则 | 字符串格式为 `字段_方向` |
| `hasStock` | `Integer` | 是否有货 | `0` 表示无货，`1` 表示有货 |
| `skuPrice` | `String` | 价格区间 | 格式为 `min_max` |
| `brandIds` | `List<Long>` | 品牌 ID 集合 | 多选品牌筛选 |
| `attrs` | `List<String>` | 属性筛选 | 格式为 `attrId_值1:值2:值3` |
| `pageNum` | `Integer` | 页码 | 后端为 0 基页码 |
| `pageSize` | `Integer` | 页大小 | 控制每页返回数量 |

补充约定：

- `pageNum` 在后端是 0 基页码。前端 UI 若显示“第 1 页”，请求时必须传 `0`。
- `sort` 推荐支持：
  - `skuPrice_asc`
  - `skuPrice_desc`
  - `saleCount_desc`
  - `hotScore_desc`
- `skuPrice` 支持以下格式：
  - `_6000`：6000 以下
  - `3000_`：3000 以上
  - `3000_6000`：3000 到 6000
- `attrs` 需要按属性 ID 和属性值拼接，例如：
  - `2017506484590100481_iPhone Air:iPhone 17e`

### 2.4 响应数据定义

搜索结果来自 `ProductSearchResult`，前端重点消费以下字段：

| 字段 | 类型 | 说明 | 页面用途 |
| --- | --- | --- | --- |
| `products` | `List<Product>` | 商品列表 | 首页推荐区、搜索结果列表 |
| `total` | `Long` | 总记录数 | 搜索结果统计 |
| `pages` | `Integer` | 总页数 | 分页器 |
| `pageNum` | `Integer` | 当前页码 | 分页回显 |
| `brands` | `List<RelatedBrand>` | 相关品牌 | 搜索页品牌筛选 |
| `attrs` | `List<RelatedAttr>` | 相关属性 | 搜索页属性筛选 |
| `catalogs` | `List<RelatedCatalog>` | 相关分类 | 搜索页分类筛选 |

商品列表中的 `Product` 可用字段：

- `skuId`
- `spuId`
- `skuTitle`
- `skuPrice`
- `skuImg`
- `saleCount`
- `hasStock`
- `hotScore`
- `brandId`
- `catalogId`
- `brandName`
- `brandImg`
- `catalogName`
- `attrs`

筛选聚合字段说明：

- `brands` 用于渲染品牌筛选区，每项包含 `brandId`、`brandName`、`logo`
- `catalogs` 用于渲染分类筛选区，每项包含 `catalogId`、`catalogName`
- `attrs` 用于渲染属性筛选区，每项包含 `attrId`、`attrName`、`attrValue`

重要限制：

- 搜索接口内部虽然配置了 ES 高亮，但当前返回结果未将高亮内容回填到 `products` 中。
- 前端不能依赖“后端返回高亮标题”能力。
- 如果需要高亮关键字，只能在前端本地基于 `keyword` 和 `skuTitle` 做文本高亮增强。

### 2.5 请求示例

#### 首页推荐请求（空参数）

首页可使用空参数获取推荐商品，也可以显式带分页参数。

```json
{}
```

或：

```json
{
  "pageNum": 0,
  "pageSize": 12
}
```

#### 搜索请求示例

```json
{
  "keyword": "iPhone",
  "hasStock": 0,
  "catalogId": 225,
  "attrs": [
    "2017506484590100481_iPhone Air:iPhone 17e"
  ],
  "skuPrice": "_6000"
}
```

#### 基于上一次结果继续检索示例

第一次搜索：

```json
{
  "keyword": "iPhone",
  "pageNum": 0,
  "pageSize": 20
}
```

接口返回 `brands`、`attrs`、`catalogs` 后，用户继续筛选时，前端必须从上一次返回结果中选择条件并重新组装请求，例如：

```json
{
  "keyword": "iPhone",
  "brandIds": [10001],
  "catalogId": 225,
  "attrs": [
    "2017506484590100481_iPhone Air"
  ],
  "pageNum": 0,
  "pageSize": 20
}
```

## 3. 商城首页开发要求

### 3.1 页面定位

首页是商城主入口，承担以下职责：

- 承接品牌与活动氛围展示
- 提供统一搜索入口
- 展示推荐商品、热卖商品、猜你喜欢等商品区域
- 将用户引导到搜索结果页继续筛选

首页不承担复杂筛选逻辑，筛选能力主要由搜索结果页承接。

### 3.2 页面结构

首页至少包含以下模块：

1. 顶部快捷导航区
2. Logo 与主搜索区
3. 热词区
4. 购物车入口
5. 主视觉区（Banner / 活动位）
6. 左侧类目入口
7. 推荐商品区
8. 热卖商品区
9. 猜你喜欢区

布局要求：

- 采用 PC 居中宽屏布局，建议主内容宽度约 `1200px`
- 头部、主视觉、商品楼层区分层明确
- 信息密度偏高，接近京东首页的节奏

### 3.3 搜索入口要求

首页顶部必须提供主搜索框：

- 支持输入关键字并提交
- 支持回车搜索
- 支持点击搜索按钮搜索
- 提交后跳转到搜索结果页，并携带 `keyword`

热词区要求：

- 搜索框下方展示热词或推荐词
- 若当前无接口提供热词，可先使用静态文案占位

### 3.4 推荐商品区要求

首页进入后立即请求搜索接口获取推荐商品：

- 默认请求可传空参数
- 推荐使用 `pageNum = 0`
- 推荐使用 `pageSize = 12` 或 `16`

推荐商品区展示要求：

- 首屏至少展示两行商品卡片
- 商品卡片包含：
  - 商品主图
  - 商品标题
  - 商品价格
  - 销量
  - 品牌名
- 无货商品允许展示，但状态需要弱化

### 3.5 热卖与猜你喜欢区要求

可复用同一搜索接口，按不同排序策略获取商品：

- 热卖：推荐传 `sort = ["saleCount_desc"]`
- 猜你喜欢：可继续使用空参数，或传 `sort = ["hotScore_desc"]`

如果当前版本不拆多次请求，也可复用首页首屏推荐结果按前端分区展示，但文案上要保持“推荐”“热卖”“猜你喜欢”的区块感。

### 3.6 商品卡片交互要求

- 鼠标悬停时增加阴影、描边或轻微抬升效果
- 点击卡片需预留跳转商品详情页能力
- 商品标题过长时允许两行截断
- 价格信息需要明显强调
- 无货商品展示“暂时无货”或相近状态文案

## 4. 搜索结果页开发要求

### 4.1 页面定位

搜索结果页负责承接以下流程：

- 从首页搜索框进入的关键字检索
- 页面内再次输入关键字检索
- 根据上一次返回的相关品牌、属性、分类持续缩小检索范围
- 支持排序、分页、库存筛选、价格筛选

### 4.2 页面结构

搜索结果页至少包含以下区域：

1. 顶部搜索栏
2. 搜索结果统计区
3. 已选筛选条件区
4. 分类筛选区
5. 品牌筛选区
6. 属性筛选区
7. 排序工具栏
8. 商品列表区
9. 分页区

布局要求：

- 搜索栏固定在页面顶部主区域
- 筛选区可放在列表上方，也可做左侧筛选栏
- 商品列表区为主要视觉焦点
- 分页器位于列表底部

### 4.3 搜索与持续检索要求

搜索结果页必须维护两份核心状态：

1. 当前查询参数状态
2. 最近一次成功返回的聚合结果状态

持续检索规则：

- 首次进入页面后，根据当前关键字或路由参数发起搜索
- 用户进行二次筛选时，只能从最近一次成功返回的 `brands`、`attrs`、`catalogs` 中选择条件
- 每次请求成功后，前端必须用最新返回的聚合结果刷新筛选面板
- 已选条件需要独立维护，避免刷新筛选面板后丢失当前已选状态

前端实现要求：

- 品牌筛选：从 `brands` 中读取 `brandId`
- 分类筛选：从 `catalogs` 中读取 `catalogId`
- 属性筛选：从 `attrs` 中读取 `attrId`、`attrValue`
- 重新请求时：
  - 选中品牌后写入 `brandIds`
  - 选中分类后写入 `catalogId`
  - 选中属性值后写入 `attrs`

### 4.4 筛选规则

#### 分类筛选

- 分类筛选使用 `catalogId`
- 默认按单选处理
- 用户切换分类时，覆盖旧分类并重新搜索
- 支持取消已选分类

#### 品牌筛选

- 品牌筛选来源于本次结果中的 `brands`
- 使用 `brandIds` 数组传参
- 前端可支持多选品牌
- 每次变更品牌选择后立即重新搜索

#### 属性筛选

- 属性筛选来源于本次结果中的 `attrs`
- 每个属性组显示属性名与可选值
- 选中后按 `attrId_值1:值2:值3` 格式写入 `attrs`

示例：

- 选择属性 ID `2017506484590100481`
- 选择属性值 `iPhone Air`
- 则参数为：`2017506484590100481_iPhone Air`

如果同一属性支持多选，则同一属性下多个值使用冒号拼接。

#### 库存筛选

- 提供“仅看有货”开关
- 勾选后传 `hasStock = 1`
- 若需要支持“仅看无货”，则传 `hasStock = 0`
- 取消筛选后不传该字段

#### 价格筛选

- 提供预设价格区间选项
- 提供自定义价格区间输入
- 前端必须按 `min_max` 格式组装 `skuPrice`

### 4.5 排序要求

搜索结果页至少支持以下排序方式：

- 综合：不传 `sort`
- 价格从低到高：`["skuPrice_asc"]`
- 价格从高到低：`["skuPrice_desc"]`
- 销量优先：`["saleCount_desc"]`
- 热度优先：`["hotScore_desc"]`

交互要求：

- 当前排序项要有明确高亮状态
- 切换排序时重置到第一页
- 切换排序时保留当前关键字与所有筛选条件

### 4.6 分页要求

分页必须遵守后端 0 基页码规则。

UI 与请求转换规则：

- UI 第 1 页，对应请求 `pageNum = 0`
- UI 第 2 页，对应请求 `pageNum = 1`
- 以此类推

分页交互要求：

- 翻页时保留所有筛选条件
- 翻页时保留排序状态
- 翻页时保留关键字
- 切换筛选或排序时，分页重置到 UI 第 1 页

### 4.7 已选条件区要求

页面必须展示当前已生效的查询条件，建议以标签形式展示：

- 关键字
- 已选分类
- 已选品牌
- 已选属性
- 库存筛选
- 价格区间

交互要求：

- 每个标签可单独取消
- 取消后立即重新搜索
- 提供“清空筛选”入口
- 清空筛选后保留当前关键字，回到仅关键字搜索状态

### 4.8 商品列表区要求

商品列表需要展示以下信息：

- 商品图片
- 商品标题
- 商品价格
- 销量
- 品牌名
- 分类名（可选显示）
- 库存状态

交互要求：

- 商品标题支持关键字本地高亮（可选增强）
- 无货商品需要明显标识
- 点击商品进入商品详情页
- 鼠标悬停时有明显交互反馈

### 4.9 页面状态要求

#### 加载中状态

- 首次搜索时显示骨架屏或列表 loading
- 切换筛选、排序、分页时显示局部 loading
- 避免页面整体闪烁

#### 空结果状态

当 `total = 0` 时：

- 展示空状态文案
- 提供“清空筛选”入口
- 可提供返回首页或查看推荐商品入口

#### 接口失败状态

- 展示错误提示
- 保留上一次成功结果
- 不直接清空商品列表

#### 慢响应与重复点击

- 对筛选、排序切换做防抖或节流
- 若连续触发多次请求，仅以最后一次请求结果为准

### 4.10 路由与状态回显要求

为了保证刷新后页面可恢复，建议将以下状态同步到路由参数或可恢复状态中：

- `keyword`
- `pageNum`
- `sort`
- `catalogId`
- `brandIds`
- `attrs`
- `hasStock`
- `skuPrice`

刷新页面后要求：

- 搜索框回显关键字
- 已选筛选项可恢复
- 列表可按当前条件重新请求

## 5. 视觉与交互风格要求

### 5.1 视觉方向

整体风格参考京东 PC 商城：

- 主色建议使用京东红系，例如 `#e1251b`
- 页面背景使用浅灰与白色分层
- 核心内容区采用居中布局
- 模块之间有清晰边界，避免过度留白
- 商品信息密度高于普通品牌官网风格

### 5.2 重点视觉元素

- 搜索框要明显、醒目，具备首页主入口地位
- 价格信息使用红色强调
- 促销信息、按钮、重点状态使用强对比色
- 筛选栏、排序栏、分页器要有清晰分隔与层级

### 5.3 交互动效

- 商品卡片 hover 增加阴影、边框或轻微位移
- 筛选项 hover 与选中状态需有明显区分
- 排序切换、分页切换应有明确反馈
- 不需要复杂动画，以快速响应和清晰反馈为主

### 5.4 响应式要求

本次开发为 PC 优先，移动端仅要求基础适配：

- 窄屏下允许筛选区域折叠
- 搜索栏和商品列表需保证基础可用
- 不要求单独设计完整 H5 专版页面

## 6. 前端状态管理建议

为了保证持续检索逻辑稳定，建议前端至少维护以下状态：

- `searchParams`：当前请求参数
- `searchResult`：最近一次成功搜索结果
- `selectedFilters`：已选筛选项
- `loading`：加载状态
- `error`：错误状态

建议行为：

- 所有筛选操作都基于 `searchParams` 派生新请求
- 所有筛选面板都基于 `searchResult.brands`、`searchResult.attrs`、`searchResult.catalogs` 渲染
- 当接口失败时，不覆盖 `searchResult`

## 7. 验收标准

开发完成后，页面至少满足以下验收要求：

1. 首页首次进入后，能够使用空参数成功请求并展示推荐商品。
2. 首页搜索框提交后，能够跳转到搜索结果页并携带关键字。
3. 搜索结果页能够根据上一次返回的品牌、属性、分类继续缩小检索范围。
4. 搜索结果页在筛选、排序、翻页后，能够正确保留当前上下文。
5. 已选筛选项能够回显、单独取消、整体清空。
6. 空结果、接口失败、无货商品三类状态均有明确展示。
7. 前端分页展示为 1 基，接口请求为 0 基，二者转换正确。
8. 文档中所有字段名、接口路径、参数格式与当前后端实现保持一致。

## 8. 测试场景

### 8.1 首页场景

1. 首次进入首页，发送空参数请求并展示推荐商品。
2. 首页切换到热卖分区时，使用 `sort = ["saleCount_desc"]` 获取商品。
3. 点击首页搜索按钮后，跳转到搜索结果页并回显关键字。

### 8.2 搜索页场景

1. 搜索 `iPhone`，列表正确展示。
2. 使用上一次返回的品牌筛选继续检索，`brandIds` 组装正确。
3. 使用上一次返回的分类筛选继续检索，`catalogId` 组装正确。
4. 使用上一次返回的属性筛选继续检索，`attrs` 格式正确。
5. 切换价格区间 `_6000`、`3000_6000`、`3000_`，参数正确。
6. UI 切到第 2 页时，请求实际传 `pageNum = 1`。
7. 切换排序后，分页重置到第一页，其他筛选条件保留。
8. 清空筛选后，仅保留关键字重新搜索。

### 8.3 异常场景

1. 搜索结果为空时，页面展示空状态与清空筛选入口。
2. 接口失败时，页面展示错误提示并保留上一次结果。
3. 返回商品 `hasStock = false` 时，页面正常展示无货状态，不影响列表渲染。
4. 刷新页面后，关键字和筛选条件能够恢复并重新发起请求。

## 9. 默认约定与边界说明

- 本次文档默认面向 PC 页面开发，移动端仅做基础适配说明。
- 首页推荐商品不新增独立推荐接口，直接复用现有搜索接口。
- 搜索页的筛选项来源仅依赖每次搜索结果中的聚合字段，不依赖额外品牌、分类、属性接口。
- 当前后端未直接返回高亮标题，前端不得将后端高亮作为必需依赖。
- 类目入口、热词、Banner 若暂时无独立接口，可先以静态数据占位，但搜索主链路必须使用现有接口完成。
